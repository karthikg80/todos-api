Elevating the Everything View

Three approaches to transform a flat task list into a rich, scannable workspace

Currently: flat <ul> grouped by category, 4 chips max per row, single density mode, no switchable groupings.

A

Layered Enhancement (Recommended)

What: Add a Group By dropdown + Density toggle to the existing list. Enrich each row with color-coded priority borders, inline progress rings for subtasks, and smarter chip layout. Keep the current renderTodos() pipeline — extend it, don't replace it.

Pros

  • Lowest risk — builds on working rendering pipeline
  • Each piece (grouping, density, visual cues) ships independently
  • CSS-driven density modes — no DOM changes for compact/expanded
  • Group-by logic already partly exists (category grouping in "All")

Cons

  • String-template rendering gets more complex
  • Doesn't lay groundwork for reuse in other views yet
B

Component Extraction

What: Extract a TaskListRenderer module that owns grouping, density, and row rendering. Wire it into the Everything view first, but architect it for reuse across Today, Upcoming, and project views later.

Pros

  • Clean separation — rendering logic gets its own module
  • Switchable grouping becomes a first-class concept
  • Future views (Today, Upcoming) can reuse the same renderer

Cons

  • Bigger refactor of renderTodos() in filterLogic.js
  • Risk of breaking existing view-specific rendering (Today's section labels, Upcoming's date headers)
  • More upfront work before visible results
C

Virtual List + Rich Cards

What: Replace the DOM-heavy <ul> with a virtual scrolling list (custom or lightweight lib). Each task becomes a "card" with expanded metadata, mini charts, and rich interactions. Full redesign of the task row.

Pros

  • Handles 1000+ tasks smoothly (current DOM approach doesn't)
  • Most dramatic visual improvement
  • Enables features like inline Gantt bars, sparklines

Cons

  • Largest scope — touches drag-drop, inline editing, keyboard nav
  • May need a dependency (or significant custom code)
  • Breaks existing test selectors and accessibility patterns
